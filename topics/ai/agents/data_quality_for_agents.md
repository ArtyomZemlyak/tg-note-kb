# Качество данных в обучении агентов ИИ: Подход LIMI и Принцип Эффективности Агентности

## Описание

Качество данных играет критически важную роль в обучении агентов искусственного интеллекта. Подход LIMI (Less is More for Agency) особенно подчеркивает важность высококачественных обучающих примеров по сравнению с большими, но менее качественными наборами данных. Работа LIMI формулирует Принцип эффективности агентности (Agency Efficiency Principle): сложная машинная автономия возникает не из-за обилия данных, а благодаря стратегическому отбору демонстраций, которые отражают суть агентного поведения.

## Принципы высококачественных данных для агентов ИИ

### 1. Актуальность
- Данные должны отражать реальные сценарии использования агента
- Примеры должны быть репрезентативными для задач, которые агент будет решать в практике

### 2. Качественные траектории
- Траектории должны демонстрировать полные процессы решения задач
- Должны показывать эффективное использование инструментов
- Должны включать правильное планирование и принятие решений
- Должны отражать рассуждения, использование инструментов и обратную связь от среды

### 3. Тщательная аннотация
- Примеры должны быть тщательно аннотированы экспертом
- Процесс создания траекторий должен включать человеческий контроль
- Высококачественные траектории должны быть отобраны из многих попыток

### 4. Стратегическая курация
- Вместо масштабирования данных, фокус на стратегическом отборе минимального набора высококачественных примеров
- Ориентация на конкретные домены, требующие полного спектра агентных способностей
- Использование синтезированных запросов из реальных сценариев (например, GitHub Pull Requests)

## Подход LIMI к качеству данных

### Ограничение количества, акцент на качестве
- Вместо тысяч примеров, использование небольшого количества (в случае LIMI - 78 примеров) высококачественных данных
- Акцент на тщательно созданных траекториях решения задач
- Использование взаимодействия человека и ИИ для создания качественных примеров
- Средняя длина траектории 42.4k токенов, самая длинная - 152k токенов

### Методология сбора
- Сотрудничество между аннотаторами и GPT-5 до достижения успешного решения задачи
- Систематический синтез запросов из профессиональных сценариев и GitHub Pull Requests
- Фокус на задачах из реальной практики (Vibe Coding и Research Workflows)
- Захват полного рабочего процесса, включая итерации доработки, исправления ошибок и стратегической адаптации

## Принцип эффективности агентности (Agency Efficiency Principle)

Сложная машинная автономия возникает не из-за обилия данных, а благодаря стратегическому отбору демонстраций, которые отражают суть агентного поведения. Этот принцип фундаментально меняет дорожную карту перехода от «думающего ИИ» к надёжному «работающему ИИ».

## Результаты подхода LIMI
- Улучшение метрик на AgenceBench с 45% до 73.5% благодаря качественным данным
- 53.7 процентных пункта улучшения по сравнению с моделями, обученными на 10 000 примерах
- Использование в 128 раз меньше данных при достижении превосходных результатов
- Демонстрация, что количество не всегда коррелирует с эффективностью
- Подтверждение важности человеческого вмешательства в процесс создания обучающих данных
- Стабильные преимущества в задачах на обобщение (инструменты, программирование, научные вычисления)

## Практические рекомендации

- Всегда фокусироваться на качестве, а не только на количестве обучающих примеров
- Использовать процесс "человек в цикле" для создания высококачественных траекторий
- Рассматривать длину и подробность траекторий при создании обучающих данных
- Оценивать эффективность агентов на реалистичных бенчмарках, таких как AgenceBench
- Смещать фокус с вычислительной «грубой силы» на интеллектуальное мастерство инженерии высококачественных данных
- Стратегически отбирать минимальный набор высококачественных примеров, отражающих суть агентного поведения
