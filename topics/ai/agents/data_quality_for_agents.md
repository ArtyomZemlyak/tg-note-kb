# Качество данных в обучении агентов ИИ: Подход LIMI

## Описание

Качество данных играет критически важную роль в обучении агентов искусственного интеллекта. Подход LIMI (Less is More for Agency) особенно подчеркивает важность высококачественных обучающих примеров по сравнению с большими, но менее качественными наборами данных.

## Принципы высококачественных данных для агентов ИИ

### 1. Актуальность
- Данные должны отражать реальные сценарии использования агента
- Примеры должны быть репрезентативными для задач, которые агент будет решать в практике

### 2. Качественные траектории
- Траектории должны демонстрировать полные процессы решения задач
- Должны показывать эффективное использование инструментов
- Должны включать правильное планирование и принятие решений

### 3. Тщательная аннотация
- Примеры должны быть тщательно аннотированы экспертом
- Процесс создания траекторий должен включать человеческий контроль
- Высококачественные траектории должны быть отобраны из многих попыток

## Подход LIMI к качеству данных

### Ограничение количества, акцент на качестве
- Вместо тысяч примеров, использование небольшого количества (в случае LIMI - 78 примеров) высококачественных данных
- Акцент на тщательно созданных траекториях решения задач
- Использование взаимодействия человека и ИИ для создания качественных примеров

### Методология сбора
- Сотрудничество между аннотаторами и GPT-5 до достижения успешного решения задачи
- Средняя длина траектории 42.4k токенов, самая длинная - 152k токенов
- Фокус на задачах из реальной практики (Vibe Coding и Research Workflows)

## Результаты подхода LIMI
- Улучшение метрик на AgenceBench с 45% до 73% благодаря качественным данным
- Демонстрация, что количество не всегда коррелирует с эффективностью
- Подтверждение важности человеческого вмешательства в процесс создания обучающих данных

## Практические рекомендации

- Всегда фокусироваться на качестве, а не только на количестве обучающих примеров
- Использовать процесс "человек в цикле" для создания высококачественных траекторий
- Рассматривать длину и подробность траекторий при создании обучающих данных
- Оценивать эффективность агентов на реалистичных бенчмарках, таких как AgenceBench
