# FunSearch: Система для поиска функций с использованием LLM

## Общее описание

FunSearch - это система для автоматического открытия научных алгоритмов, представленная в Nature в 2023 году. Система сочетает большие языковые модели (LLM) с эволюционным поиском для открытия эффективных алгоритмов, особенно в задачах комбинаторной оптимизации. FunSearch была одной из первых систем, которая убедительно продемонстрировала эффективность подхода LLM-эволюции в научных открытиях, открыв новые алгоритмы, которые превзошли существующие решения в задачах, таких как покрытие множеств и упаковка в корзину.

## Архитектура и подход

FunSearch использует инновационный подход, основанный на следующих компонентах:

1. **Программирование с подсказками**: система использует подсказки, чтобы направить LLM на генерацию функций, которые могут быть частью более крупного алгоритма, а не на решение задачи целиком.

2. **Разделение функций**: задача разбивается на составные функции, которые эволюционируются отдельно.

3. **Оценка через симуляцию**: сгенерированные функции оцениваются в контексте выполнения полной задачи с использованием симуляции.

4. **Эволюционный отбор**: лучшие функции сохраняются и используются для генерации новых вариантов.

## Ключевые инновации

### 1. Программирование с подсказками

Вместо того чтобы просить LLM решить задачу полностью, FunSearch фокусируется на генерации конкретных функций, которые могут улучшить существующий алгоритм. Это позволяет системе быть более управляемой и фокусированной.

### 2. Использование существующих структур

Система использует существующие программные каркасы и фреймворки, в которые встраиваются эволюционированные функции, что повышает надежность и интерпретируемость результатов.

### 3. Многофазная эволюция

FunSearch использует многофазный подход к эволюции, где разные фазы могут фокусироваться на разных аспектах задачи или на разных уровнях абстракции.

## Экспериментальные результаты

FunSearch достигла впечатляющих результатов в нескольких научных задачах:

- **Упаковка в корзину**: открытие новых эвристик, которые превзошли существующие алгоритмы
- **Покрытие множеств**: создание новых функций, которые улучшили эффективность решения
- **Комбинаторная оптимизация**: демонстрация применимости подхода к различным классам задач

## Сравнение с другими системами

### FunSearch vs AlphaEvolve
- Обе системы используют LLM для эволюции алгоритмов
- FunSearch фокусируется на программировании с подсказками, в то время как AlphaEvolve использует более прямой подход эволюции кода
- FunSearch показала конкретные улучшения в задачах комбинаторной оптимизации

### FunSearch vs DeepEvolve
- DeepEvolve расширяет возможности путем добавления глубоких исследований внешних источников
- DeepEvolve использует шестиступенчатый итеративный процесс, в то время как FunSearch использует более простую структуру
- DeepEvolve имеет компонент автоматической отладки, который отсутствует в FunSearch

## Ограничения и вызовы

- **Зависимость от качества подсказок**: результаты сильно зависят от качества начальных подсказок и структуры задачи
- **Ограниченная область применения**: первоначально разработана для задач комбинаторной оптимизации
- **Вычислительные затраты**: эволюционный процесс может быть ресурсоемким

## Значение и влияние

FunSearch сыграла важную роль в развитии автоматического открытия алгоритмов:

- **Демонстрация возможностей**: убедительно показала, что LLM могут открывать действительно новые и эффективные алгоритмы
- **Новый подход к программированию**: ввела концепцию программирования с подсказками для научного открытия
- **Влияние на будущие системы**: стала основой для разработки более сложных систем, таких как DeepEvolve
