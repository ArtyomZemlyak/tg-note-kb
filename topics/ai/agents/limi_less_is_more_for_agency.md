# LIMI (Less is More for Agency): Парадигма качества данных в агентах ИИ

## Описание

LIMI (Less is More for Agency) представляет собой подход к разработке агентов ИИ, подчеркивающий принцип "меньше — значит больше". Вместо использования огромных объемов данных, LIMI фокусируется на высококачественных обучающих примерах, чтобы достичь максимальной эффективности агентов. Этот подход доказывает, что качество данных решает, а не их количество.

## Основные результаты

- Улучшение метрик на AgenceBench с 45% до 73% при дообучении (fine-tuning) на 78 высококачественных примерах
- Демонстрация важности качества данных для разработки способных агентов ИИ
- Повышение эффективности агентов с помощью тщательно курируемых обучающих траекторий

## Домены исследования

Исследование охватывает два основных домена:

### 1. Vibe Coding
- Реальные запросы из практики программирования
- Синтетические запросы из GitHub Pull Requests
- Задачи, связанные с кодированием и разработкой программного обеспечения

### 2. Research Workflows
- Научные рабочие процессы
- Задачи, связанные с исследовательской деятельностью
- Использование инструментов и планирование в научной среде

## Методология сбора данных

### Качественные траектории
- Для каждого запроса аннотатор и GPT-5 работали вместе в SII CLI до тех пор, пока задача не была успешно решена
- Из всех созданных траекторий выбирались лучшие
- Средняя длина траектории составила 42.4 тысячи токенов
- Самая длинная траектория достигла 152 тысяч токенов

### Сбор запросов
- 60 реальных запросов из практики
- Синтетические запросы, извлеченные из GitHub Pull Requests

## Подход к дообучению (Fine-tuning)

- Модель дообучалась на собранных данных
- Фокус на использовании инструментов и траекториях выполнения задач
- Ограниченное количество обучающих примеров (всего 78 примеров)

## Ключевые инсайты

1. **Качество важнее количества**: Небольшое количество высококачественных примеров может превзойти большие, но менее качественные наборы данных
2. **Тщательная курация данных**: Важно не просто собирать данные, а тщательно отбирать и аннотировать их
3. **Использование инструментов**: Успешные агенты требуют обучающих примеров, демонстрирующих эффективное использование инструментов
4. **Длинные траектории**: Длинные и подробные траектории решений могут значительно улучшить способности агента

## Применение на практике

LIMI представляет собой важный сдвиг в парадигме разработки агентов ИИ, подчеркивая, что:

- Меньше, но качественнее данных может привести к лучшим результатам
- Ручная аннотация и курация могут быть более эффективными, чем автоматическое масштабирование
- Длинные и подробные траектории решения задач критически важны для обучения агентов
