# Over-squashing в графовых нейронных сетях

## Описание

Over-squashing (чрезмерное сжатие) - это фундаментальная проблема в глубоких графовых нейронных сетях (GNN), при которой информация из экспоненциально растущего рецептивного поля узла сжимается в вектор фиксированного размера, создавая узкое место. Эта проблема ограничивает способность GNN эффективно обрабатывать сложные дальнодействующие зависимости в графах.

## Два режима over-squashing

Согласно статье "gLSTM: Mitigating Over-Squashing by Increasing Storage Capacity", проблему over-squashing можно разделить на два различных режима сбоя:

### 1. Over-squashing чувствительности

- **Описание**: традиционно изучаемая проблема низкой чувствительности, при которой выход узла перестает реагировать на изменения в удаленных узлах
- **Причина**: сбой распространения сигнала, когда информация не может эффективно передаваться между удаленными узлами
- **Математическое описание**: выражается убывающей нормой Якобиана
- **Решения**: топологические методы, перестройка графа, улучшение потока информации

### 2. Over-squashing емкости

- **Описание**: насыщение представления узла фиксированного размера, которому просто не хватает места для хранения всей необходимой информации из его окрестности
- **Причина**: ограничение в емкости хранения, даже при идеальном потоке сигнала
- **Математическое описание**: проявляется как сбой избирательной чувствительности, где GNN теряет способность различать релевантные и нерелевантные узлы
- **Решения**: архитектурные инновации, такие как gLSTM, увеличивающие внутреннюю память узла

## Влияние на производительность

Over-squashing существенно ограничивает возможности GNN в следующих аспектах:

- Понимание дальнодействующих взаимодействий (например, в молекулярных графах при разработке лекарств)
- Обработка сложных структур графов
- Предсказание свойств графа, требующих информации из больших расстояний

## Методы решения

### Топологические методы
- Перестройка графа (graph rewiring)
- Использование спектральных методов
- Изменение структуры графа для улучшения передачи информации

### Архитектурные методы
- Введение ассоциативной памяти (как в gLSTM)
- Увеличение емкости хранения узла
- Использование более мощных архитектур на уровне узлов

## Задачи для оценки over-squashing

- **Neighbor Associative Recall (NAR)**: синтетическая задача для измерения ограничения емкости при неглубоком графе
- Другие бенчмарки для оценки способности GNN обрабатывать дальние связи
