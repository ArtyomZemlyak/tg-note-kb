# MCP (Model Context Protocol) - Протокол Контекста Модели

## Краткое описание

MCP (Model Context Protocol) - это протокол, предназначенный для управления контекстом в системах на основе LLM, позволяющий решать проблемы ограниченного контекста моделей. Протокол переносит часть функций хранения и обработки данных на клиентскую сторону, позволяя LLM сосредоточиться на планировании и рассуждении.

## Основная проблема

Фундаментальная проблема использования LLM заключается в том, что объем контекста ограничен. Традиционный подход подразумевал LLM как мост передачи/хранения данных, совмещающий функции резонера/планировщика. MCP предлагает альтернативу: LLM теперь выполняет только роль планировщика/логического процессора, а функции хранения и обработки данных перемещаются на сторону клиента MCP.

## Ключевые концепции и особенности

- **Ограничение контекста**: Контекст LLM конечен, что создает фундаментальные ограничения для работы агентов и приложений на их основе
- **Архитектурный сдвиг**: Переход от тонкого клиента к функционалу на стороне клиента MCP для хранения и обработки данных
- **Роль LLM**: LLM теперь функционирует как резонер/планировщик, а не как мост передачи/хранения данных
- **Перенос функций хранения**: Функции хранения информации переносятся с контекста LLM на клиент MCP
- **Решение проблемы контекста**: Это подход к работе с ограничением контекста, переносящий функции хранения на клиентскую сторону

## Применение и связи с другими технологиями

- **SO/ограниченный вывод (constrained output)**: Связан с технологиями, управляющими выводом LLM
- **SGR (Selective Generation Restriction)**: Связан с методами ограничения генерации
- **Другие изощрения**: Относится ко всем методам, направленным на преодоление ограничений LLM

## Инсайты и перспективы

MCP представляет собой радикальный подход к преодолению ограничений контекста LLM, перенося функции хранения данных на клиентскую сторону. Вместо попыток расширить контекст LLM, архитектура использует MCP-клиент как надежное хранилище данных, необходимых для выполнения задач. Это позволяет LLM сосредоточиться на своих сильных сторонах - рассуждении и планировании, в то время как сложные операции по управлению контекстом выполняются на уровне протокола.
