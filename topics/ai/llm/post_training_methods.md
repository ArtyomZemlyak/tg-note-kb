# Методы постобучения LLM

## Описание

Методы постобучения - это техники, применяемые после основного этапа предобучения модели, для адаптации модели к специфическим задачам или улучшения её способности следовать инструкциям, безопасности и полезности.

## Основные методы постобучения

### Использование инструментов (MCP и синтетические)
На этапе постобучения могут использоваться тысячи инструментов для улучшения способности модели работать с внешними инструментами. В Kimi K2 использовали 3000 MCP (Model Context Protocol) инструментов с GitHub и ещё 10 тысяч синтетических инструментов.

[[ai/llm/memory/mcp_model_context_protocol.md]] - Подробное описание протокола контекста модели и его роли в интеграции инструментов

### Генерация агентов на основе инструментов
По инструментам генерируются тысячи агентов, которые создают задачи для обучения. Эти задачи оцениваются в режиме LLM-as-a-Judge, и успешные траектории становятся базой для обучения.

## Другие методы постобучения

### Инструкционное тонирование (Instruction Tuning)
Адаптация модели к выполнению инструкций с помощью специализированных датасетов.

### Обучение с подкреплением (RLHF/RLAIF)
Использование обучения с подкреплением для улучшения ответов модели на основе предпочтений или автоматических оценок.

### Прямое подгонка по предпочтениям (DPO)
Метод, который использует прямое сравнение пар ответов для обучения без использования функции вознаграждения.
