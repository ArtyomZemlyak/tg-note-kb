# Шаблоны рассуждения (Reasoning Patterns)

## Краткое описание

Шаблоны рассуждения - это фиксированные процедурные структуры решения задач, которые остаются неизменными для всех экземпляров определенного типа задач. Отличаются от адаптивных задач, где стратегия решения меняется от случая к случаю.

## Основная информация

Ключевое различие делается между двумя типами задач:

### 1. Шаблонное рассуждение (Patterned Reasoning)
Характеризуется:
- **Устойчивый процедурный шаблон решения**: Одинаковый для всех экземпляров задачи
- **Фиксированная последовательность шагов**: Постоянный подход к решению
- **Предсказуемость процесса**: Структура рассуждения заранее известна
- **Применимость PARO и SFT+RLVR**: Можно эффективно обучать с минимальными ресурсами рационалей

Примеры задач с шаблонным рассуждением:
- Классификация по фиксированным правилам
- Верификация по чек-листу
- Извлечение информации по схеме
- Финансовый анализ и классификация

### 2. Адаптивное рассуждение (Adaptive Reasoning)
Характеризируется:
- **Изменяющаяся стратегия решения**: Отличается от случая к случаю
- **Непредсказуемость процесса**: Требуется гибкий подход
- **Нуждается в большем объеме разметки**: Для охвата пространства решений
- **Не подходит для PARO**: Экономия на рационалях не работает

Примеры задач с адаптивным рассуждением:
- Олимпиадная математика
- Сложное планирование
- Соревновательное программирование

## Новые концепции и термины

- **РАционали (Rationales)**: Явные объяснения или цепочки рассуждений, генерируемые моделью
- **Развилочные токены (Forking tokens)**: Токены в цепочке рассуждений, замена которых влияет на конечный ответ
- **RFTD (Rollout-based Forking Token Detection)**: Метод выявления ключевых решений в цепочке рассуждений
- **Шаблон решения**: Фиксированная структура шагов для решения задачи

## Примеры шаблонов рассуждения

### NSM (Numerical Semantic Matching)
Шаги шаблона:
1. **Найти числа в контексте** - Извлечь числовые упоминания в документах
2. **Интерпретировать смысл** - Определить время, показатель, субъект числового значения
3. **Выровнять сущности/контекст** - Сопоставить числовые упоминания
4. **Принять решение об эквивалентности** - Определить, эквивалентны ли два числовых упоминания

### TPC (Transaction Purpose Classification)
Шаги шаблона:
1. **Извлечь ключевые признаки** - Выделить информацию из банковской транзакции
2. **Соотнести с правилами таксономии** - Сопоставить с фиксированным набором категорий
3. **Выбрать класс** - Определить соответствующую категорию транзакции
4. **Обосновать (опционально)** - Кратко пояснить классификацию

## Примеры применения

Шаблоны рассуждения особенно полезны в:
- Финансовом анализе
- Юридической обработке документов
- Медицинской диагностике по протоколу
- Технической поддержке с фиксированными процедурами
- Логистике и цепочках поставок
