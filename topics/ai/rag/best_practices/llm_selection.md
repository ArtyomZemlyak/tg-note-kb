# Выбор языковых моделей для RAG-систем

## Описание

Выбор подходящей языковой модели (LLM) критически важен для RAG-систем, так как LLM интегрирует извлеченную информацию в генерируемый ответ. Выбор модели зависит от требований к производительности, качеству ответов, доступным вычислительным ресурсам и специфике задачи.

## Базовые модели

### Qwen-2.5-7B-Instruct

- Базовая модель для экспериментов с RAG
- Хорошее соотношение качества и производительности
- Поддержка инструментов и инструкций

## Адаптированные версии

### RefalMachine/RuadaptQwen2.5-14B-Instruct

- Адаптация Qwen для русскоязычного контекста
- Улучшенное понимание русского языка
- Оптимизирована для специфичных задач

### T-lite и T-pro

- t-tech/T-lite-it-1.0 - легковесная версия для быстрых инференсов
- t-tech/T-pro-it-2.0 - более мощная версия для сложных задач
- Оптимизированы для IT-задач и документации

## Agentic RAG

### Qwen3-30B-A3B-Instruct-2507

- Модель для построения агентных RAG-систем
- Высокая способность к рассуждению
- Поддержка сложных цепочек вызовов инструментов

## Критерии выбора модели

### Производительность

- Скорость генерации
- Потребление вычислительных ресурсов
- Размер модели и требования к памяти

### Качество

- Способность к рассуждению
- Согласованность и точность ответов
- Умение работать с извлеченным контекстом

### Специфика задачи

- Языковая поддержка
- Предметная область
- Требования к безопасности и качеству

## Практические рекомендации

### Для начала

- qwen-2.5-7b-instruct как базовая модель
- Простой старт с понятной архитектурой

### Для русскоязычных систем

- RuadaptQwen2.5-14B-Instruct для лучшего понимания контекста
- Учет культурных и языковых особенностей

### Для агентных систем
