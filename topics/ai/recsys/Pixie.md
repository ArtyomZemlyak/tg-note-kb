# Pixie: Biased Random Walks для рекомендаций

## Описание

Pixie - это алгоритм рекомендаций на основе biased random walks по графу взаимодействий. Модель была разработана и применена в Pinterest для решения задач рекомендаций пинов и досок.

## Основная идея

Pixie использует концепцию случайных блужданий по графу для генерации рекомендаций:
- Блуждания стартуют из объекта (пин или доска), для которого нужны рекомендации
- В процессе блужданий считаются node visitation counts
- Чем чаще встречается вершина в блужданиях, тем выше её счет; рекомендации - топ по посещаемости

## Архитектура и алгоритм

### Структура графа

- Двудольный граф пинов и досок
- Пины и доски - вершины
- Сохранения пинов на доски - рёбра

### Biased Random Walk

- Блуждания учитывают пользователя (язык, тематика пина)
- Не равновероятные переходы: смещения (bias) на основе пользовательского контекста
- При блужданиях учитываются атрибуты узлов и пользовательский профиль

### Формирование рекомендаций

- Для рекомендаций пользователю запускаются блуждания из его сохраненных пинов
- Собирается статистика посещений
- Возвращаются топ-N узлов по частоте посещений

## Преимущества

- Простота реализации и интерпретируемости
- Возможность учета пользовательского контекста через biased переходы
- Надежность и устойчивость к шуму в данных
- Масштабируемость (до определенного уровня)

## Ограничения

- Не учитывает контент айтемов напрямую
- Ограниченная способность к обобщению на новые айтемы
- Могут возникать проблемы с длинными цепочками несвязанных рекомендаций
- Менее эффективен по сравнению с современными GNN-моделями

## Применение в Pinterest

- Использовался до появления PinSage
- Применяется в рекомендациях "Related Items (pin → pin)"
- Используется для поиска связанных пинов и досок
- Служил основой для сэмплирования соседей в PinSage

## Связь с другими моделями

- **GraphSAGE**: Pixie использовался для сэмплирования соседей в GraphSAGE (в PinSage)
- **PinSage**: использует идеи Pixie для weighted aggregation в GraphSAGE
- **Random Walk подходы**: обобщение подходов DeepWalk, node2vec для рекомендаций
