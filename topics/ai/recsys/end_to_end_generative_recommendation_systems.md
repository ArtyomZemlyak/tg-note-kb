# End-to-end генеративные рекомендательные системы: EGA v1 и развитие концепции

## Описание

End-to-end генеративные рекомендательные системы представляют собой новое направление в области рекомендательных систем, при котором используются генеративные подходы для непосредственного создания рекомендаций без традиционных стадий кандидат-генерации и ранжирования. Вместо того чтобы использовать отдельные модели для генерации кандидатов и последующего их ранжирования, end-to-end системы учатся генерировать финальные рекомендации напрямую.

## EGA v1 (Meituan)

### Описание

EGA v1 - это реализация end-to-end генеративной рекомендательной системы, представленная компанией Meituan на CIKM'25. Несмотря на то, что работа была выпущена весной того же года, из слайдов стало известно, что A/B тест проводили уже больше года назад, что свидетельствует о раннем начале внедрения таких систем в промышленной среде.

### Особенности

- **End-to-end подход**: Система генерирует рекомендации напрямую без промежуточных стадий
- **Генеративность**: Использует генеративные методы для создания рекомендаций
- **Промышленная реализация**: Реализована и протестирована в реальных условиях с огромным объемом данных

### Значимость

EGA v1 от Meituan демонстрирует, что крупные китайские компании активно инвестируют в end-to-end генеративные рекомендательные системы и достигают значительных результатов. Это подтверждает тренд, о котором говорилось на CIKM'25 - trend на end2end системы только набирает обороты.

## OneRec и Core команда

### Описание

OneRec - это еще одна реализация end-to-end генеративной рекомендательной системы, разработанная Kuaishou. Участники core команды OneRec посетили CIKM'25, и удалось узнать много деталей, которых нет в публикациях.

### Особенности

- **LLM-based подход**: OneRec использует большие языковые модели для генерации рекомендаций
- **End-to-end движок**: Не требует дальнейших стадий ранжирования, генерирует финальные рекомендации напрямую
- **Интеграция SIDs**: Использует семантические идентификаторы (Semantic IDs) для представления айтемов

### Инфраструктурные вызовы

Западные команды, вдохновленные работой OneRec, пытаются реализовать подобные системы, но сталкиваются с основными сложностями, лежащими в инфраструктуре. По опыту Яндекса, это действительно непростая часть, требующая значительных вычислительных ресурсов и оптимизации.

### Использование GPU для semantic IDs

Интересный инсайд от CIKM'25: для построения semantic IDs в OneRec использовали всего 200 GPU. Это указывает на то, что процесс токенизации айтемов и создания семантических представлений является ресурсоемким, но выполнимым с разумным количеством вычислительных мощностей.

## Сравнение с другими подходами

### Традиционные рекомендательные системы

| Аспект | Традиционная система | End-to-end генеративная система |
|--------|------------------------|--------------------------------|
| Архитектура | Многостадийная (кандидат-генерация, ранжирование) | Одностадийная (прямая генерация) |
| Обучение | Контрастивное обучение (InfoNCE) | Генеративное обучение (Next Token Prediction) |
| Инференс | Сравнение эмбеддингов | Прямая генерация |
| Сложность инфраструктуры | Умеренная | Высокая |
| Интерпретация | Логика через стадии | Логика через ризонинг (в LLM-based системах) |

### LLM-based рекомендательные системы

End-to-end генеративные системы часто используют LLM в своей основе, но отличаются от других LLM-based подходов:

- **Фокус на финальные рекомендации**: Система генерирует именно рекомендации, а не промежуточные результаты
- **Ризонинг интегрирован**: Логический вывод встроен в процесс генерации
- **Высокая вычислительная сложность**: Требует значительных ресурсов для рантайма

## Вызовы и ограничения

1. **Высокая вычислительная стоимость**: End-to-end генерация требует значительных вычислительных мощностей
2. **Инфраструктурная сложность**: Требуется сложная инфраструктура для поддержки генеративных моделей
3. **Оценка качества**: Традиционные метрики могут быть не полностью применимы к генеративным системам
4. **Время отклика**: Генерация может занимать больше времени по сравнению с традиционными подходами

## Перспективы развития

CIKM'25 показал, что:

- Крупные компании верят в end-to-end генеративные рекомендательные системы
- Эти системы уже внедряются в промышленной среде (Meituan, Kuaishou)
- Западные команды активно изучают и пытаются внедрить подобные подходы
- Основные сложности связаны с инфраструктурой, а не с архитектурой моделей
