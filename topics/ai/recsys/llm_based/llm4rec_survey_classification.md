# Классификация LLM для рекомендательных систем: Обзор

## Обзор

Данный обзор основан на обширном исследовании "How Can Recommender Systems Benefit from Large Language Models: A Survey", опубликованном в ACM Transactions on Information Systems (TOIS). В статье предложена систематическая классификация подходов использования больших языковых моделей (LLM) в рекомендательных системах.

## Классификация по этапам рекомендательной системы

Классификация систематизирует работы по тому, где именно в pipeline рекомендательной системы адаптируются LLM:

### 1.1 LLM для инженерии признаков (Feature Engineering)

#### 1.1.1 Аугментация признаков на уровне пользователей и предметов
Подходы, которые используют LLM для генерации или улучшения признаков на уровне отдельных пользователей и предметов. Эти методы позволяют извлекать богатые семантические представления из текстового описания предметов и профилей пользователей.

Примеры подходов:
- LLM4KGC: Использование LLM для генерации графов знаний для рекомендаций
- TagGPT: Генерация тегов для предметов с помощью GPT-моделей
- KAR: Knowledge-aware recommendation с использованием LLM
- PIE: Представление интересов пользователей с помощью LLM

#### 1.1.2 Генерация экземпляров (Instance-level Sample Generation)
Подходы, которые используют LLM для генерации новых обучающих экземпляров, включая синтетические взаимодействия или расширение данных.

Примеры подходов:
- GReaT: Генерация данных для задач рекомендаций
- ONCE: One-class collaborative learning через генерацию экземпляров
- AnyPredict: Предсказание на основе сгенерированных данных

### 1.2 LLM как энкодер признаков (Feature Encoder)

#### 1.2.1 Улучшение представлений (Representation Enhancement)
Методы, которые используют LLM для генерации улучшенных встраиваний для пользователей или предметов, часто заменяя традиционные методы эмбеддинга.

Примеры подходов:
- U-BERT: Использование BERT для построения пользовательских профилей
- UNBERT: Унифицированные эмбеддинги пользователей и предметов
- PLM-NR: Использование предобученных языковых моделей для нейронных рекомендаций
- NExT: Нейронная сеть с трансформерами для представлений

#### 1.2.2 Унифицированная кросс-доменная рекомендация
Подходы, которые используют LLM для построения универсальных представлений, применимых в нескольких доменах.

Примеры подходов:
- ZESRec: Zero-shot cross-domain recommendation
- UniSRec: Унифицированные представления для кросс-доменной рекомендации
- TransRec: Transferable representation learning

### 1.3 LLM как функция оценки/ранжирования (Scoring/Ranking Function)

#### 1.3.1 Задача оценки предметов (Item Scoring Task)
Подходы, которые используют LLM для оценки релевантности предметов пользователям, часто через прямое предсказание вероятностей или оценок.

Примеры подходов:
- LMRecSys: Языковые модели для рекомендательных систем
- TALLRec: Prompt-based подход с использованием больших языковых моделей
- FLAN-T5: Использование инструкционно-тюненных моделей для рекомендаций
- RecBooth: Платформа для LLM-базированных рекомендаций

#### 1.3.2 Задача генерации предметов (Item Generation Task)
Методы, которые используют LLM для генерации новых рекомендаций или создания персонализированного контента.

Примеры подходов:
- GPT4Rec: Использование GPT для рекомендаций на основе последовательностей
- VIP5: Визуальные и языковые признаки для рекомендаций
- P5-ID: Генерация ID предметов с помощью LLM
- DiffRec: Генерация рекомендаций через диффузионные модели

#### 1.3.3 Гибридная задача (Hybrid Task)
Подходы, которые комбинируют классификацию и генерацию, используя LLM для одновременного решения нескольких задач.

Примеры подходов:
- P5: Unified encoder-decoder framework
- M6-Rec: Массовая мультимодальная модель для рекомендаций
- InstructRec: Instruction-following модели для рекомендаций

### 1.4 LLM для пользовательской интеракции (User Interaction)

#### 1.4.1 Целеориентированная пользовательская интеракция
Подходы, которые позволяют пользователям задавать конкретные цели или запросы, и система рекомендует соответствующие предметы через естественный язык.

Примеры подходов:
- TG-ReDial: Генерация рекомендаций с объяснениями
- TCP: Conversational recommendation через планирование
- MESE: Emotional и эмпатическое понимание в рекомендациях

#### 1.4.2 Открытая пользовательская интеракция
Подходы, которые поддерживают открытые, продолжительные диалоги с пользователями для уточнения предпочтений.

Примеры подходов:
- BARCOR: Balanced conversational recommendation
- RecInDial: Рекомендации через открытые диалоги
- UniCRS: Унифицированная conversational recommendation system

### 1.5 LLM для управления пайплайном рекомендаций (RS Pipeline Controller)

Подходы, которые используют LLM как центральный контроллер для управления различными компонентами рекомендательной системы, включая агентов и автоматизированные решения.

Примеры подходов:
- Chat-REC: Conversational recommendation через чат-бота
- RecLLM: Управление рекомендательным процессом через LLM
- RAH: Reasoning and acting для рекомендаций
- RecMind: Использование LLM как "разума" для рекомендаций

## Значение классификации

Эта систематическая классификация позволяет:
1. Лучше понимать текущее состояние исследований в области LLM для рекомендаций
2. Идентифицировать пробелы в исследованиях
3. Выявлять перспективные направления развития
4. Сравнивать различные подходы между собой
5. Упростить доступ к научным публикациям для исследователей
