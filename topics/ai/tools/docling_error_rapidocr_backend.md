# Ошибка RapidOcrOptions в Docling: отсутствует поле backend

## Описание проблемы

Ошибка `convert_document_into_docling_document: Unexpected error: "RapidOcrOptions" object has no field "backend"` возникает при использовании библиотеки Docling для обработки документов с включенным OCR. Эта проблема указывает на то, что в используемой версии библиотеки поле `backend` было изменено, удалено или никогда не существовало в классе `RapidOcrOptions`.

Термин `backend` в контексте OCR-библиотек обычно относится к внутреннему движку распознавания, который используется для обработки изображений документов. В случае RapidOCR это может быть ссылка на конкретную реализацию OCR-движка.

## Причины возникновения

1. **Изменение API**: Разработчики Docling могли изменить структуру класса `RapidOcrOptions` в новой версии
2. **Использование устаревшего кода**: Примеры или документация могут содержать устаревшие параметры
3. **Несовместимость версий**: Используется несовместимая версия библиотеки с существующим кодом
4. **Неправильная конфигурация**: Попытка доступа к несуществующему полю в RapidOcrOptions
5. **Изменения в интеграции RapidOCR**: Параметры RapidOCR были переименованы или реорганизованы внутри Docling

## Решение проблемы

Для исправления этой ошибки, попробуйте следующие подходы:

1. **Проверьте документацию**: Ознакомьтесь с официальной документацией текущей версии Docling
2. **Альтернативные параметры**: Используйте вместо `backend` другие доступные параметры:
   - `ocr_options` - для настройки параметров OCR
   - `engine` - если таковой параметр существует для выбора OCR-движка
   - `ocr_mode` или `ocr_method` - для выбора метода OCR
3. **Обновление кода**: Обновите свой код в соответствии с текущим API
4. **Проверка версии**: Убедитесь, что вы используете совместимую версию библиотеки
5. **Проверка конфигурации**: Убедитесь, что вы правильно создаете объект настроек OCR

## Пример фикса

Вместо:
```python
from docling.datamodel.document import RapidOcrOptions

options = RapidOcrOptions(backend="tesseract")  # или другой бэкенд
```

Используйте:
```python
from docling.datamodel.document import OCRConfig

# Проверьте доступные параметры в документации
# Возможно, нужно использовать OCRConfig с другими параметрами
config = OCRConfig(do_ocr=True)  # или другие доступные параметры
```

## Сравнение с другими OCR-библиотеками

В других OCR-библиотеках выбор бэкенда осуществляется по-разному:
- Tesseract: использует параметр PSM (Page Segmentation Mode) для настройки метода распознавания
- EasyOCR: позволяет выбирать библиотеки через параметр `recognition_model`
- PaddleOCR: позволяет настраивать движки через конфигурационные файлы
