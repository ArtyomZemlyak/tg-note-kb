# Инженерия подсказок для ИИ-кодирования (Prompt Engineering for Coding)

## Краткое описание

Инженерия подсказок для ИИ-кодирования - это набор методов и подходов для эффективной структуризации запросов к ИИ-ассистентам кодирования. Это позволяет получать более точные, контекстуально подходящие и качественные результаты при взаимодействии с ИИ-моделями.

## Основные принципы инженерии подсказок

### 1. Ясность и специфичность
- Использование ясного, недвусмысленного языка при описании задач кодирования
- Предоставление достаточного контекста о проекте, архитектуре и требованиях
- Четкое определение ожидаемого формата и структуры выходного кода
- Четкое определение проблемы и желаемого результата

### 2. Структурированный подход
- Разбиение сложных задач кодирования на логические компоненты
- Использование структурированных форматов, таких как списки, заголовки и разделители
- Применение шаблонов для стандартизации общих задач кодирования

### 3. Примеры и шаблоны
- Предоставление примеров желаемых паттернов кода, стилей или структур
- Использование шаблонных подсказок для общих сценариев кодирования
- Реализация few-shot обучения, показывая примеры перед основным запросом

## Расширенные техники

### Цепочка рассуждений (Chain of Thought) в генерации кода
1. **Цепочка рассуждений**: Поощрение модели "мыслить вслух", разбивая сложные проблемы кодирования на промежуточные шаги
2. **Самосогласованность (CoT-SC)**: Генерация нескольких разных подходов к одной задаче кодирования и выбор лучшего решения
3. **Синтез цепочки рассуждений (CaT)**: Новый подход, который синтезирует лучший ответ из частей несовершенных ответов

### Гибридный подход SGR (Structured Generation Response) + Инструменты
- Использование отдельного инструмента `generate_reasoning`, вызываемого сразу после пользовательского ввода
- Внутри рассуждения, использование методологии SGR с секциями:
  - Цель
  - План
  - Ограничения
  - Проверки предположений
  - Сигналы о том, нужны ли вызовы инструментов

## Шаблоны для ИИ-ассистентов кодирования

### Шаблон генерации кода
```
Как опытный разработчик, создайте код [КОНКРЕТНОГО ТИПА] для [ФУНКЦИОНАЛЬНОСТИ]. Код должен:
- Следовать принципам [КОНКРЕТНОЙ АРХИТЕКТУРЫ/ПАТТЕРНА]
- Включать надлежащую обработку ошибок для [КОНКРЕТНЫХ СЦЕНАРИЕВ]
- Быть совместимым с [КОНКРЕТНЫМ СТЕК ТЕХНОЛОГИЙ]
- Включать модульные тесты в соответствии с конвенциями [ТЕСТИРОВОЙ ФРЕЙМВОРКИ]
- Поддерживать согласованность с существующими руководствами по [КОНКРЕТНОМУ СТИЛЮ]
```

### Шаблон рецензирования кода
```
Проверьте следующий код на:
1. Качество кода и сопровождаемость
2. Уязвимости безопасности
3. Соображения производительности
4. Консистентность архитектуры
5. Соблюдение лучших практик

Особое внимание уделите [КОНКРЕТНЫМ ОБЛАСТЯМ БЕСПОКОЙСТВА] и предложите улучшения для [КОНКРЕТНЫХ АСПЕКТОВ].
```

## Архитектурные соображения

### Безопасная архитектура для помощников кодирования
- Использование изолированных песочниц
- Контроль доступа к сети
- Безопасное управление учетными данными
- Применение ограничений по частоте и предотвращению злоупотреблений
