# Шаблон правил для кодассистентов (rules_template)

## Краткое описание

Шаблон правил (rules_template) - это комплексный и оптимизированный шаблон пользовательских подсказок (правил) для ИИ-ассистентов кодирования, таких как Cursor, CLINE, RooCode и Windsurf. Основная цель шаблона - улучшить производительность ИИ-кодирования за счет объединения памяти, логического рассуждения в подзадачи и лучших практик подсказок.

## Основные особенности

### 1. Комбинированный подход
- **Память** - Постоянная документация проекта
- **Логика подзадач** - Структурированные рабочие процессы
- **Лучшие практики подсказок** - Принципы программной инженерии

### 2. Кроссплатформенная совместимость
- Cursor: `.cursor/rules/`
- RooCode: `.roo/` 
- CLINE: `.clinerules/`
- Использование символических ссылок для единого источника правды

### 3. Система документации
- Структурированные файлы документации для постоянного контекста
- Автоматическое обновление документации после изменений
- Разделение на требование продукта, архитектуру, технические спецификации

## Файловая структура

### Основные файлы правил
- `rules.mdc` - Общие правила и руководящие принципы
- `plan.mdc` - Рабочий процесс планирования с логической цепочкой рассуждений
- `implement.mdc` - Рабочий процесс реализации на основе принципов программной инженерии
- `debug.mdc` - Правила отладки для устранения неполадок
- `memory.mdc` - Система документирования для постоянного контекста
- `directory-structure.mdc` - Руководство по модульной организации проекта

### Файлы системы памяти
- `product_requirement_docs.md` - Требования и цели проекта
- `architecture.md` - Документация системной архитектуры
- `technical.md` - Технические спецификации
- `tasks/tasks_plan.md` - Отслеживание задач и прогресса проекта
- `tasks/active_context.md` - Текущий контекст разработки

## Рабочий процесс из 5 фаз

1. **Требования и уточнения** - Четкий сбор требований
2. **Исчерпывающий поиск и оптимальное планирование** - Поиск оптимальных решений
3. **Проверка пользователем** - Проверка плана с пользователем
4. **Реализация** - Итеративная разработка с тестированием
5. **Дальнейшие предложения** - Оптимизация и предложения функций

## Установка и использование

### Базовая установка
1. Копирование соответствующего каталога правил в корень проекта:
   - Cursor: `.cursor/rules/`
   - RooCode: `.roo/`
   - CLINE: `.clinerules/`

### Расширенная установка
2. Создание настраиваемых режимов для экономии токенов:
   - **Режим чата**: Для разговоров ИИ без операций с файлами
   - **Режим записи**: Для чтения, записи и выполнения команд
   - **Режим MCP**: Для операций с сервером MCP

## Целевая аудитория

- **Начинающие**: Дружелюбное введение в ИИ-кодирование с структурированным подходом
- **Опытные пользователи**: Расширенные функции и оптимизация токенов для продвинутых пользователей
