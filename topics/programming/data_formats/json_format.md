# Формат JSON (JavaScript Object Notation)

## Описание

**JavaScript Object Notation (JSON)** - это текстовый формат для хранения и передачи данных, основанный на синтаксисе объектов JavaScript. Это один из самых популярных форматов обмена данными в веб-разработке и API.

## Структура

JSON строится на двух структурах:
1. Коллекция пар ключ-значение (объект, словарь, хеш-таблица)
2. Упорядоченный список значений (массив, вектор, список)

## Типы данных

JSON поддерживает следующие типы данных:
- **Числа** (целые и с плавающей точкой)
- **Строки** (в двойных кавычках)
- **Логические значения** (true или false)
- **Объекты** (неупорядоченная коллекция пар ключ-значение)
- **Массивы** (упорядоченная коллекция значений)
- **null** (отсутствие значения)

## Пример

```json
{
  "users": [
    { "id": 1, "name": "Alice", "role": "admin" },
    { "id": 2, "name": "Bob", "role": "user" }
  ],
  "count": 2,
  "active": true
}
```

## Преимущества

- Универсальный формат, поддерживаемый большинством языков программирования
- Читаемый человеком
- Интегрирован в JavaScript
- Легко парсится и генерируется

## Недостатки

- Более многословный по сравнению с другими форматами (см. [[programming/data_formats/toon_format.md]])
- Нет встроенной поддержки комментариев
- Нет специальных типов для дат
- Меньше эффективен по использованию токенов в LLM

## Сравнение с TOON

Формат [[programming/data_formats/toon_format.md]] был разработан для решения проблемы эффективности использования токенов в LLM и обеспечивает 30-60% сокращение количества токенов по сравнению с JSON для табличных данных.

См. также: [[ai/llm/optimization/toon_for_llm_optimization.md]] - оптимизация LLM с использованием альтернативных форматов данных.
