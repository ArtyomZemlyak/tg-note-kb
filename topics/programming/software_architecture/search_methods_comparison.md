# Сравнение методов поиска: Infini-gram, BM25 и SPLADE

## Обзор

Современные системы поиска и извлечения информации используют различные подходы для эффективного поиска и ранжирования релевантных документов. Три важных метода - Infini-gram, BM25 и SPLADE - представляют разные философии и технические решения для информационного поиска.

## Infini-gram

### Основная концепция
Infini-gram - это система для точного поиска n-грамм в сверхбольших текстовых корпусах, использующая FM-Index. Был представлен в статье "Infini-gram mini: Exact n-gram Search at the Internet Scale with FM-Index", ставшей лучшей статьей на EMNLP2025.

### Особенности
- Использует FM-Index (Full Text Minute Index) для сжатия и индексации
- Позволяет обрабатывать n-граммы практически неограниченной длины
- Обеспечивает точный поиск (а не вероятностный)
- Размер индекса составляет всего 44% от исходного корпуса
- 18x быстрее при индексации, 3.2x меньше использования памяти

### Применение
- Точный поиск в петабайтных текстовых корпусах
- Фильтрация больших датасетов
- Возможное применение в RAG-системах и системах памяти для агентов

## BM25

### Основная концепция
BM25 (Best Match 25) - вероятностная модель оценки релевантности, используемая в поисковых системах. Улучшенная версия модели TF-IDF, которая лучше учитывает длину документа и частоту терминов.

### Особенности
- Учитывает частоту терминов (TF) и обратную частоту документов (IDF)
- Нормализует по длине документа
- Включает параметры k1 и b для настройки насыщения частоты и влияния длины документа
- Прост в реализации и вычислительно эффективен

### Применение
- Поисковые системы
- Ретриверы в ИИ-системах
- Информационный поиск

См. также: [[bm25_search_algorithm.md]]

## SPLADE

### Основная концепция
SPLADE (Sparse Lexical and Expansion Model) - это подход к поиску, который создает разреженные представления документов с использованием нейронных сетей. SPLADE генерирует веса терминов для документов, создавая разреженные векторы, которые могут содержать не только оригинальные термины, но и их семантические расширения.

### Особенности
- Создает разреженные векторные представления документов
- Объединяет лексические и семантические аспекты поиска
- Позволяет учитывать синонимы и семантически связанные термины
- Обеспечивает интерпретируемость результатов поиска

### Применение
- Современные поисковые системы
- Гибридные методы поиска
- Сценарии, где важна интерпретируемость

## Сравнительный анализ

| Критерий | Infini-gram | BM25 | SPLADE |
|----------|-------------|------|--------|
| Тип поиска | Точный (exact match) | Вероятностный | Разреженный векторный |
| Техническая основа | FM-Index, суффиксные массивы | TF-IDF с улучшениями | Нейронные сети |
| Длина запроса | Практически неограниченная | Ограниченная | Ограниченная |
| Простота реализации | Средняя | Высокая | Низкая |
| Требования к памяти | Экономичные (44% от корпуса) | Средние | Зависит от размера эмбеддингов |
| Скорость поиска | Высокая | Высокая | Средняя-высокая |
| Интерпретируемость | Высокая | Высокая | Средняя |

## Интеграция и гибридные подходы

В современных системах ИИ все чаще используются гибридные подходы, которые могут объединять преимущества различных методов:

1. **Infini-gram + BM25**: Использование точного поиска n-грамм для начального отбора, затем ранжирование с помощью BM25
2. **BM25 + SPLADE**: Гибридный поиск, сочетающий лексические и семантические аспекты
3. **Infini-gram + SPLADE**: Точный поиск для определенных типов запросов, семантический для других

## Практические применения

### RAG-системы
- Infini-gram может быть полезен для точного поиска фрагментов текста
- BM25 обеспечивает классическую релевантность
- SPLADE добавляет семантическое понимание

### Системы памяти агентов
- Infini-gram может использоваться для точного поиска в архивных данных
- BM25 для быстрого ретривала по релевантности
- SPLADE для поиска с учетом семантики

## Перспективы развития

Новое поколение систем поиска может использовать все три подхода в зависимости от типа запроса и требований к системе. Infini-gram, в частности, открывает возможности для работы с петабайтными корпусами данных, что особенно актуально в эпоху больших языковых моделей, требующих доступа к обширным текстовым базам.
