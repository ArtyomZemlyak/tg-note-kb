# Системы контроля версий (Version Control Systems)

## Краткое описание

Системы контроля версий (VCS) - это программные системы, которые позволяют отслеживать изменения в файлах и управлять историей этих изменений. Они особенно важны при разработке программного обеспечения, где требуется координация работы нескольких разработчиков, сохранение истории изменений и возможность возврата к предыдущим версиям кода.

## Основные концепции

### 1. Репозиторий (Repository)
- Хранилище всех файлов проекта и их исторических версий
- Может быть локальным или удаленным
- Содержит метаданные о версиях и изменениях

### 2. Коммит (Commit)
- Фиксация изменений в репозитории с описанием
- Уникально идентифицируется хэшем
- Создает новую точку в истории проекта

### 3. Ветвление (Branching)
- Возможность создания независимых линий разработки
- Позволяет работать над разными функциями параллельно
- Обычно есть главная ветка (main/master) и вспомогательные

### 4. Слияние (Merging)
- Объединение изменений из разных веток
- Может происходить автоматически или требовать разрешения конфликтов
- Сохраняет полную историю изменений

## Распространенные системы

### Git
- Распределенная система контроля версий
- Наиболее популярна в современной разработке
- Поддерживает сложные схемы ветвления и слияния

### SVN (Subversion)
- Централизованная система контроля версий
- Более простая модель, но с ограничениями в распределенной работе

### Mercurial
- Распределенная система, альтернатива Git
- Простота в освоении

## Основные команды Git

### Инициализация и клонирование
- `git init` - создание нового репозитория
- `git clone <url>` - копирование удаленного репозитория

### Работа с изменениями
- `git add <файл>` - добавление файлов к следующему коммиту
- `git commit -m "сообщение"` - создание коммита
- `git status` - проверка состояния репозитория

### Ветвление и слияние
- `git branch` - список веток
- `git checkout <ветка>` или `git switch <ветка>` - переключение между ветками
- `git merge <ветка>` - слияние указанной ветки с текущей

### Работа с удаленным репозиторием
- `git push` - отправка изменений на удаленный репозиторий
- `git pull` - получение и слияние изменений из удаленного репозитория

### Управление синхронизацией
- `git stash` - временное сохранение изменений при необходимости синхронизации
- [[git_synchronization_errors.md]] - Решение ошибок синхронизации, включая "Repository has uncommitted changes"

### Структура репозитория
- Четкая организация файлов и директорий
- Использование .gitignore для исключения временных файлов

## Инструменты и платформы

### GitHub

- Интеграция с различными инструментами разработки

### GitLab
- Альтернативная платформа с функциями CI/CD

- Поддержка DevOps-процессов
