# Проблемы "вайб-кодинга" и деградация архитектуры в AI-ассистированной разработке

## Общее описание

Статья "ПОЧЕМУ ВАЙБ-КОДИНГ НЕ РАБОТАЕТ" Михаила Шогина обсуждает проблемы с AI-ассистированной разработкой ("vibe-coding"), при которой разработчики полагаются на AI-агентов, таких как Claude и Cursor, для генерации кода. Начальные впечатляющие результаты быстрой генерации большого количества кода со временем сменяются деградацией архитектуры.

## Симптомы архитектурной деградации

- **Замедление работы агентов** с течением времени
- **Избыточная генерация кода** для простых задач
- **Архитектурные анти-паттерны**, такие как функции, вызывающие одну и ту же функцию обработки несколько раз без необходимости
- **Функции с чрезмерным количеством параметров**
- **Циклические зависимости** между пакетами
- **Дублирование бизнес-логики**
- **God объекты** и ненужные уровни абстракции

## Систематический подход к решению

Автор предлагает систематический подход с использованием формальных описаний архитектуры и автоматической валидации:

### Архитектура как графы

- **Структурные (статические) графы**: показывают потенциальные отношения компонентов, построенные из анализа AST (абстрактного синтаксического дерева)
- **Поведенческие (динамические) графы**: показывают реальные пути выполнения, построенные из трасс тестов приемки (acceptance test traces)

### Инструмент archlint

Автор представляет свой инструмент "archlint" для автоматического построения и проверки архитектурных графов из Go кода, направленный на предотвращение деградации архитектуры в AI-ассистированной разработке.

## Значимость

- Важное понимание ограничений и проблем современных подходов к AI-ассистированной разработке
- Предлагает конкретное решение для поддержания качества архитектуры при использовании AI-инструментов
- Подчеркивает необходимость баланса между продуктивностью, предоставляемой AI, и архитектурной целостностью системы
